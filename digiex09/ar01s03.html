3.1. Introduction

Dave Edenfield, W8RIT, will collect reports from each district as well as additional monitoring stations.

3.2. Contents of Report

Transmitting stations will report station city or township, software used, radio used, antenna used, interface used, sound card used and power output.
Receiving stations will report receiving station city, software used, radio used, antenna used, interface used, sound card used and percentage of characters received correctly.
A receiving station may report only some modes if all modes are not available to that station.
Stations using homebrew interfaces should provide a short description of the interface.

3.3. Delivery of Report

A blank text file with the schedule will be made available on the ARPSC web site. Paste your copy into the appropriate place in the text file and email the result to W8RIT.
Prior to the exercise, download the reporting form from http://www.mi-arpsc.org/downloads/DigitalExerciseReport.txt.
Email the report to: w8rit@arrl.net.

3.4. Publication of Results

Results, including analysis, will be published at: http://www.mi-arpsc.org/Exercise_09spring.php.
